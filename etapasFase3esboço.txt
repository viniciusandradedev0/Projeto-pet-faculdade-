# Objetivo: substituir localStorage por banco real + autenticação JWT
# Projeto: backend/PawsPlace.Api  |  .NET 8  |  SQLite (dev)

# ============================================================
# FASE 2 — BACK-END
# ============================================================

- [x] Etapa 8 — Setup do projeto ASP.NET Core
  - Web API .NET 8 criada (dotnet new webapi)
  - Pacotes: EF Core Sqlite 8.0.15, BCrypt.Net-Next, JwtBearer 8.0.15
  - Models: Usuario, Animal, Pedido, Favorito
  - AppDbContext com índices únicos e relacionamentos
  - Program.cs: JWT + CORS + migrations automáticas
  - Migration inicial: CriarTabelasIniciais

- [x] Etapa 9 — Seed dos animais + verificação do banco
  - Data/Seed.cs criado com os 15 animais hardcoded
  - dotnet run confirmou: migration rodou + banco SQLite criado (56KB)

- [x] Etapa 10 — Autenticação JWT
  - JwtService: gera token HS256 com claims (NameIdentifier, Email, Name)
  - AuthService: cadastro (bcrypt work=12) + login com mensagem genérica
  - POST /api/auth/cadastro → 201 + TokenResponseDto
  - POST /api/auth/login    → 200 + TokenResponseDto
  - Erros: 409 email duplicado, 401 credenciais inválidas

- [x] Etapa 11 — Endpoints de animais (público)
  - GET /api/animais        → lista ordenada por espécie+nome
  - GET /api/animais/{slug} → 200 ou 404

- [x] Etapa 12 — Endpoints de pedidos (autenticado)
  - POST /api/pedidos       → 201, bloqueia pedido duplicado pendente (409)
  - GET  /api/pedidos/meus  → lista do usuário logado, mais recente primeiro

- [x] Etapa 12.5 — Favoritos (autenticado)
  - POST   /api/favoritos/{animalSlug} → 201 ou 409 se já existe
  - DELETE /api/favoritos/{animalSlug} → 204 ou 404
  - GET    /api/favoritos/meus         → lista com dados do animal

- [x] Etapa 13 — Endpoints de perfil (autenticado)
  - GET    /api/usuarios/me → 200 com PerfilResponseDto
  - PUT    /api/usuarios/me → valida nome+telefone, 200 ou 400
  - DELETE /api/usuarios/me → 204 (cascade remove pedidos + favoritos)

- [ ] Etapa 14 — Migração do front-end para consumir a API
  - Criar scripts/api.js (URL base da API)
  - Reescrever: auth.js, adotar.js, meus-pedidos.js, perfil.js, data.js
  - Reescrever: proteger-rota.js (verificar JWT expirado)
  - Adicionar: favoritos no front (coração nos cards + favoritos.html)
  - Remover: storage.js (substituído por fetch + JWT)
  - Manter:   validacao.js, filtros.js, render.js, tema.js, animacoes.js

- [ ] Etapa 15 — Deploy do back-end
  - Opções gratuitas: Railway, Render, Azure (estudante)
  - Variáveis de ambiente: JWT_SECRET, CONNECTION_STRING
  - CORS: aceitar origem do GitHub Pages
  - Front: apontar para URL da API em produção
